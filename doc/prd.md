# MyDiary 需求规格说明书 (PRD)

## 1. 项目概述
MyDiary 是一个旨在极简记录日常和心情的日记系统。其核心目标是让用户可以快速、直白地在多端记录每天的心情（Emoji 形式）与日记正文，并支持多用户独立使用。

## 2. 系统核心功能
### 2.1 用户系统
- **账号注册与登录**：支持多用户注册账号和登录，各个用户的日记数据相互隔离且安全。

### 2.2 日记记录与管理
- **编写与编辑日记**：用户可以为某天记录日记。编辑器支持 **Markdown 语法**，并支持富文本的图文混排和图片上传功能。
- **历史修改**：对已发表过的日记，支持二次编辑与覆盖更新。
- **日历与列表视图**：
  1. **日历视图**：提供类似打卡的直观日历看板，方便追踪哪些天有写日记。
  2. **列表视图**：瀑布流的方式查看最近日记摘要。

### 2.3 心情记录
- **心情关联**：在记录日记时同步选取 Emoji 表示当天心情。
- **可视化展示**：在日历单元格及日志列表中，把心情图标与日记醒目展示。

## 3. 技术路线
1. **前端**：基于 `Vue 3` + `TypeScript` 核心。采用 `Tailwind CSS` 进行极速样式开发，并融入 `Vue Router` 及状态管理 `Pinia`。
2. **后端**：采用 `Node.js` + `Express` 构建稳定高效的 REST API 提供服务。
3. **数据库**：
   - 采用 `PostgreSQL` 作为中心化存储。
   - **摒弃传统 ORM**，采用原生 SQL（使用 `pg` 驱动）进行直接查询操作，保障最优性能与灵活性。
